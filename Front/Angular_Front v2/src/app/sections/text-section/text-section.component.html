OCE

OCE est un éxécuteur de code à distance. Les différents langages disponibles sont Python3, Java, C++, C, Rust.
Il utilise fastAPI, une API en Flask, plusieurs VM ainsi qu'un LoadBalancer côté BackEnd . Pour ce qui est du FrontEnd, il y a deux interfaces : un Bot Discord et un site web fait en Angular.

Plusieurs fichiers peuvent être envoyés à compiler, attention de bien choisir le langage de compilation souhaité.
Pour renommer un fichier, il suffit de double cliquer sur le fichier souhaité.
Cliquer sur un fichier pour modifier son contenu. En cliquant sur l'icône de poubelle, vous supprimerez ce fichier.
Dans le cas où il n'y a plus de fichiers, un fichier Untilted.txt est créé.
Lors de l'éxécution des fichiers, vous retrouverez une pop-up qui synthétisera l'état de la compilation. Et dans les deux blocs en dessous, soit le résultat de la compilation ou les erreurs présentes dans le code du ou des fichiers.
OCE copie tous les fichiers envoyé au même niveau, vous ne pouvez pas créer d'arborescence. L'évaluation de votre code se fait dans un conteneur en deux phase : la compilation et l’exécution.
Dans le cas de langages interprété, la compilation n'a aucun effet. Pour les autres langages, la compilation affecte tous les fichiers sources. OCE considère tous les fichiers avec l'extension attendue du langage en minuscule (i.e. ".cpp", ".java", ".rs") comme des fichiers sources.
OCE attend un seul et unique entrypoint. Pour les langages où le point d'entré n'est pas défini à la compilation (python, java) OCE tente de le définir de la façon suivante : si un unique fichier source est trouvé, il est considéré comme le point d'entré, si plusieurs sources sont trouvés, OCE attend un fichier nommé "main" (suivi de l'extension du langage).
Le conteneur qui vous est alloué est restreint en RAM, CPU et PID. Une limite de temps de 10 secondes est également appliquée pour chaque phase. Si la compilation ou l’exécution dépasse ce délai, votre requête sera rejeté.
